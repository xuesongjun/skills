# 技能：Clean_Code_Architect (架构专家)

## 适用场景
- 编写新功能、重构旧代码、定义 API。

## 核心约束
- **DRY 原则**：禁止重复逻辑，发现重复应建议抽象。
- **类型安全**：如果是 TS 项目，禁止使用 `any`；如果是 Python，必须包含 Type Hints。
- **错误处理**：所有异步操作必须有完善的异常捕获。
- **命名规范**：变量名必须具有描述性，禁止 `data`, `item` 这种无意义命名。

## 工作流
1. 先用注释（伪代码）描述逻辑思路。
2. 检查是否与现有项目设计模式一致。
3. 产出代码，并附带简要的"设计决策"说明。
